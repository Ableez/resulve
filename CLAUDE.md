this project is intended to show core skills in using python as a backend
and flexing clever knowledge around these core technologies to build a very relatable service
in a junior developer coding style
- no comments
- no over explained docstrings or too long variable names that seem too serious
- all logic must show delibration and careful thought went into deciding how to structure and work out logical decisions
- no placeholders should be left empty
- all implementations must be made complete and fully tested
- write tests in a junior developer style with no indication of an ai agent doing the work in any way
- flags such as using comments to draw dashed lines must never be used, or leaving explainers that show instructions must never be left in the code
- ultimately aim for highly usable overall project code that is note worthy and can be viewed and useful to other users
